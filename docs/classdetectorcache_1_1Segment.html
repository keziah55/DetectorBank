Construct a segment for use by a slidingbuffer::SlidingBuffer.

Note that the SlidingBuffer entity is in fact a result_t* containing the address of the first result of the channel at the analysis-block sample-time origin. The analysis block is stored in a contiguous 2D array of result values. Conseqently the size parameter in particular refers to the number of channels rather than the number of samples, or number of samples per channel. Refer to the extended documentation for detectorcache::DetectorCache for the bigger picture.

Parameters
sizeNumber of entities in this segment (i.e. the number of channels).
originThe ordinal key of the first channel pointer stored.
spThe producer for this segment.
